# Unlock.ps1 レビュー メモ

## 総評

Unlock.ps1 はかなり「動けばいい」寄りの道具だが、その中でも筋はちゃんとある。

焦点は性能ではなく、
- 責務の明快さ
- 壊れ方の美しさ
- 後始末の丁寧さ

にある。

端的に言えば、実務道具として十分良い。ただし、鋭い処理をしているぶん、後片付けと説明をもう少し整えるとかなり締まる。

## 良い点

### 1. 目的が非常に明快

処理の流れが素直。

- 非破壊でコピーを作る
- 古い `.xls` は直接パッチ
- `.xlsm` / `.xlam` は一度 Excel8 形式へ落としてからパッチして戻す
- 最後に手動で完全解除する案内を出す

やることを最短距離でやっている。読む側が迷わない。

### 2. `Find-DPB` と `Patch-XlsFile` の切り出しが良い

小さいが責務が明快。

特に `Patch-XlsFile` を独立させているので、
- `.xls`
- 変換後 `.xls`

の両方に同じ処理を使える。小さい道具としてきれい。

### 3. 非破壊で `output/` にコピーを作る

Unlock のような鋭い処理は、元ファイルを触らないだけでかなり印象が変わる。ここはちゃんと押さえられている。

### 4. 「完全解除は VBE で手動」と案内している

実務的で良い。成功したふりをしないのは重要。

## 気になる点

### 1. COM オブジェクトの後始末が少し弱い

`$excel` は finally で `Quit` と `ReleaseComObject` をしているが、`$wb` の解放が見えない。

`Close` はしていても、COM 参照として残る可能性がある。Excel 自動化では、こういう残り香が後でプロセス居残りになる。

対処案:
- `$wb` も finally で `ReleaseComObject`
- `$wb = $null`
- 必要なら `GC.Collect()` / `WaitForPendingFinalizers()`

Unlock は頻繁に回す道具ではないので、少し過剰なくらいでちょうどよい。

### 2. `$tempXls` の名前が秒単位タイムスタンプで少し弱い

同時実行や短時間連打で衝突余地がある。

対処案:
- GUID を足す

性能への影響はほぼなく、事故余地だけ消せる。

### 3. `.xlsm` / `.xlam` の処理が少し「儀式」寄り

今の流れ:

- `Open -> SaveAs(tempXls) -> Close`
- `Patch -> Open(tempXls) -> SaveAs(copyPath) -> Close`

これは理解できるが、途中失敗時の `$wb` 状態が少し読み取りにくい。

catch/finally で全体を囲っているので致命的ではないが、局所的な責務としては少し濁る。

対処案:
- `Convert-ToLegacyXls`
- `Restore-FromLegacyXls`

のように一段関数で切る

Unlock という処理自体が儀式的なので、コード側はもう少し分解したほうが美しい。

### 4. `Find-DPB` の低レベル処理に意図説明が欲しい

`$data[$pos + 2] = 0x78` が何を意味するか、コードだけでは少し分かりにくい。

この手の短い低レベルコードは、短いほど説明が要る。

対処案:
- 1行コメントを足す

例:
- `DPB=` の保護フラグを `DPx=` に変えることでパスワード保護を無効化する

無言の魔術は、後で自分でも怪しく見える。

### 5. `Patch-XlsFile` の戻り値は今は良いが少し雑

文字列で
- `patched`
- `not_found`
- `error`

のように返すのは読みやすいが、将来的に状態が増えると少し雑に見える。

ただし今の規模では大きな問題ではない。現状維持でもよい。

### 6. README 上の位置づけは補助ツール寄りがよい

実装の問題ではないが、`Extract / Analyze / Diff` と同列の主線ツールとしては少し温度差がある。

対処案:
- README 上では補助ツール寄りに見せる

診断キットの横に置くには、少し刃物。

## 設計の美学として見るポイント

Unlock のような危うい処理では、成功経路よりも
- 失敗経路
- 後片付け
- 処理意図の説明

を丁寧にしたほうが美しい。

つまり、
「この道具は危うい処理をしているからこそ、後始末と説明を丁寧にする」

ここが一段良くなるポイント。

## ひとことで言うと

- Unlock.ps1 は小さい道具としてかなり筋がいい
- 非破壊、責務明快、処理の流れも素直
- ただし COM 解放と、低レベルパッチ意図の説明はもう少し整えたい

今の Unlock はちゃんと動く道具。
そこから一段良くするなら、成功経路より失敗経路と後片付けを美しくする。

茶器が鋭いぶん、洗い方まで綺麗にしておくともっと美しい。

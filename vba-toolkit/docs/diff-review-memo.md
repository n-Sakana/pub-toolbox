# Diff.ps1 レビュー メモ

## 総評

Diff.ps1 はかなり良い。小さい道具だが、責務が明快で、使い道もぶれていない。

特に良い点:
- 入口でファイルを解決し、`Get-AllModules` でモジュール集合に正規化している
- モジュール単位で `added / removed / modified / unchanged` を先に整理している
- HTML の見せ方が実務的で、目が滑りにくい

VBA の差分としてかなり正しい粒度を取れている。

## 良い点

### 1. 責務が明快

`Get-AllModules` で Diff に必要な形へ早めに寄せているので、その後のロジックがすっきりしている。可読性が高い。

### 2. モジュール差分を先に扱う構成

行差分の前にモジュール単位の added / removed / modified / unchanged を整理しているため、レビューの視線が自然。

### 3. HTML ビューアの実務感覚が良い

- サイドバーでモジュール選択
- 左右比較
- 色分け
- 変更のあるものを先に見る

派手ではないが、ちゃんと使える diff になっている。

## 気になる点

### 1. `Get-LcsDiff` という名前と実体がずれている

これは厳密な LCS ではなく、100 行先までを見る greedy な近似。

実装としては理解できるし、大きい VBA 向けの妥協として筋は通っている。ただ、名前が LCS だと読む側の期待がずれる。

対処案:
- `Get-GreedyDiff` のように名前を寄せる
- あるいはコメントで「軽量近似 diff」と明記する

「賢い近似 diff」であることを、もっと正直に名乗ったほうが美しい。

### 2. changed 判定が少し乱暴

今の `Get-LcsDiff` は、
- 一致行を前方探索で見つける
- その間の removed / added を機械的に `changed` に寄せる

という構成になっている。

特に `pairCount = Min(remCount, addCount)` はかなり機械的で、「本当に対応する変更行か」は見ていない。

そのため、
- 行の入れ替え
- ブロック移動
- 順序差が重要な変更

で、意味の薄い `changed` が増える可能性がある。

## 変更検知の改善方針について

### 結論

VBA では順番が重要なことが多いので、**changed 判定は厳格でよい**。

ただし、
- 一致点探索の精度は上げてよい
- 表示の補助は賢くしてよい

この二段に分けるのがよい。

### 整理

「変更検知を広く取る」と「変更判定を changed にまとめる」は別。

望ましい方向は以下。

- 検知は広く
- changed 判定は狭く
- 表示は親切に

### VBA で順番が効く場面

- 宣言位置
- 初期化順
- イベント発火順
- 条件分岐の並び
- 副作用のある処理順

このため、removed / added を安易に対応づけて changed にすると、順序差の重大さが薄まる危険がある。

### changed にしてよいもの

- 空白差だけ
- 大文字小文字差だけ
- コメント差だけ
- かなり高い類似度で、意味の連続性が明白なもの

### changed に寄せないほうがよいもの

- 別位置に現れた同一行
- 少し似ているだけの条件式
- 近いが意味が変わる代入文
- ブロック移動

### 改善案

#### 1. 一致点探索の精度を上げる

- 小さいモジュールでは探索窓を広げる
- 差分が続いたら探索窓を広げる
- `Sub` / `Function` / `Property` 境界をアンカーにする

#### 2. changed 化の条件は厳格にする

- 完全一致、またはかなり高い類似度のときだけ `changed`
- 少しでも怪しければ `removed` / `added` のまま残す

#### 3. 表示を親切にする

- 近接する removed / added を同じ塊として見せる
- ブロック境界を見せる
- 前後数行の文脈を見せる

順序が大事なら、無理に意味づけするより観測しやすくしたほうがよい。

## 可読性・設計面の改善案

### 1. `foreach ($name in $allNames)` の本体が少し長い

removed / added / unchanged / modified の4分岐を全部その場で処理しているので、流れは追えるが密度が高い。

対処案:
- `New-RemovedModuleDiff`
- `New-AddedModuleDiff`
- `New-UnchangedModuleDiff`
- `New-ModifiedModuleDiff`

のように分ける。

### 2. HTML 生成と差分ロジックが近接しすぎている

`sidebarSb`、`contentSb`、CSS、txt report が同じファイルに並んでいるため、差分ロジックとビュー生成が少し混ざって見える。

対処案:
- `Build-DiffSidebar`
- `Build-DiffContent`
- `Get-DiffCss`

くらいに分ける。

### 3. diff.txt が薄い

現在は要約しか出していない。

せめて以下は欲しい。
- 追加モジュール一覧
- 削除モジュール一覧
- 変更モジュール一覧

HTML が本体でも、txt を出す以上は最低限の一覧性がほしい。

## ひとことで言うと

- Diff.ps1 はかなり良い
- ただし `Get-LcsDiff` は実際には greedy 近似なので、もっと正直に名乗ったほうがよい
- VBA では順番が重要なので、changed 判定は厳格でよい
- 改善するなら「一致点探索を賢く、changed 判定は狭く、表示は親切に」がよい

今は味はいいのに、茶葉の説明札だけ少し曖昧。
その札を整えると、かなり美しい diff になる。

# Extract.ps1 レビュー メモ

## 総評

Extract.ps1 はかなり良い。端的に言えば、「入口としての役割」がはっきりしていて、使い勝手も悪くない。茶葉を広げて観察できる状態まで、素直に持っていけている。

特に良いのは、ファイル/フォルダ両対応で、`output/` を自動生成し、`combined.txt` まで出していること。単なる `.bas` の吐き出しで終わらず、モジュール一覧と統合ソースまで作るので、「まず中身をざっと見る」用途にはかなり向いている。`output/` 配下を再帰対象から除外しているのも実務的。

ただし、複数ファイル処理の設計は今のままだと危ない。ここはかなりはっきり問題がある。

## 良い点

### 1. 入口ツールとしての役割が明快

- ファイル/フォルダ両対応
- `output/` 自動生成
- モジュール別ファイル出力
- `combined.txt` 生成

「まず中身を見る」ための導線が素直で良い。

### 2. `Get-AllModuleCode -StripAttributes` の使い方が良い

属性行を落として、レビューしやすい形に寄せている。ここは実務感覚がある。

### 3. `combined.txt` の module index が実務向き

モジュール分類と行数一覧を先に出しているので、統合ソースの目次としてかなり助かる。

### 4. `output/` の再帰除外が実務的

自分の出力を次回の入力に巻き込まない。地味だが大事。

## 気になる点

### 1. 複数ファイル時に `modules/` が衝突する

現在は実行全体で単一の `outDir` を作り、その中の `modules/` に全ファイルのモジュールを書いている。

そのため、複数ファイルに同名モジュールがあると普通に上書きされる。

例:
- A.xlsm の `Module1.bas`
- B.xlsm の `Module1.bas`

後から処理したほうが残る。これはかなりまずい。抽出ツールで中身が混ざるのは、茶缶のラベルを入れ替えるのと同じで、入口の信頼を落とす。

### 2. `combined.txt` も実行全体で一つしかない

複数ファイル処理時は、最後のファイルの内容だけが残る。

### 3. `allFiles = Get-ChildItem $modulesDir -File` が構造的に危ない

`modulesDir` 全体を見ているため、前のファイルのモジュールまで
- module index
- total lines
- combined source

に混ざる可能性がある。

ここは構造的に危ない。

## 最優先で直したいこと

一番先に直すべきは、複数ファイル時の出力構造。

### 対処案

#### 案1. ファイルごとにサブフォルダを切る

例:
- `output/.../A/modules/...`
- `output/.../A/combined.txt`
- `output/.../B/modules/...`
- `output/.../B/combined.txt`

これが一番自然。抽出結果はファイル単位で閉じたほうが美しい。

#### 案2. ファイル名プレフィックスを付ける

例:
- `A__Module1.bas`
- `B__Module1.bas`
- `A_combined.txt`
- `B_combined.txt`

最低限の衝突回避にはなるが、見た目は少し落ちる。

結論としては、前者のほうがよい。

## 可読性・設計面の改善案

### 1. `combined` 生成部分が少し長い

今の `combined` 生成部分は、
- モジュール分類
- 行数集計
- index 出力
- 本文連結

まで一つの塊になっている。

やっていることは単純だが、少し密度が高い。

対処案:
- `Build-ModuleIndex`
- `Build-CombinedSource`

くらいに分ける。

今は急須の中で茶葉と湯が少し近すぎる。

### 2. ファイルごとの所要時間が累積になっている

`Write-VbaResult` に渡している時間が全体 Stopwatch の累積時間になっているので、ファイルごとの所要時間ではない。

機能的には大した問題ではないが、ログの意味として少し曖昧。

対処案:
- ファイル単位の Stopwatch を別に持つ

## ひとことで言うと

- Extract.ps1 は単体利用ではかなり良い
- 入口ツールとしての役割も明快
- ただし複数ファイル時の出力構造は早めに直したほうがいい

味はいいです。ただ、今は複数の茶葉を同じ缶に戻してしまう設計になっている。それだけは優しくない。そこを直せば、かなり安心して使える Extract になります。
